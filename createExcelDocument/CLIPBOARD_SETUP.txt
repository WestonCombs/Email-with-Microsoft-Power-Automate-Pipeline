Open File Location -> File Explorer /select (automated on Windows)
===================================================================

Minimal test workbook
---------------------

To verify VBA + ini + Python without the full orders sheet, run (from python_files):

  python createExcelDocument/create_clipboard_test_xlsm.py

That writes `email_contents/clipboard_test.xlsm`. Open File Location uses an in-workbook `#` link
so Excel does not open a file; the real `file:///…` URI is in hidden column **AC** (29).
Enable macros, click the blue **Open File Location** in B2; File Explorer should open with
that file selected (the ini path used for the test).

What runs automatically
-----------------------

On **Windows**, when you run createExcelDocument (or the full pipeline):

1. If `orders_template.xlsm` is missing, the script tries to **create it** using Excel
   in the background (COM automation). Default location:

     <BASE_DIR>/email_contents/orders_template.xlsm

   (Override with `EXCEL_TEMPLATE_PATH` in `.env` if needed.)

2. Each run writes **`excel_clipboard_launch.ini`** under **python_files** by default:

     <python_files>/excel_clipboard_launch.ini

   It lists `python.exe`, `copy_email_path_to_clipboard.py`, and **`trackingLinkViewer/tracking_link_viewer.py`**
   as `VIEWER=`. The workbook stores the **full path** to that ini in hidden cell **AA1**, so the macro
   finds it even though `orders.xlsm` lives under `email_contents`.

   Override with `EXCEL_CLIPBOARD_INI_PATH` in `.env`.

3. The workbook is saved as **`orders.xlsm`** (under `email_contents` by default) when
   the template exists. Open File Location cells use **in-workbook `#` hyperlinks** only; the real
   `file:///…` path is in hidden column **AC** (29).

4. **Tracking links (long URLs):** Up to **15** URLs per row from JSON ``tracking_links`` are stored in
   **hidden** columns **AD** through **AR** (30…44). There are no per-link cells in the visible grid;
   use **View Link List** (below) to open them.

   **Unhide URL columns (optional):** Select **AD:AR** in the **Name Box**, press Enter, then
   right‑click → **Unhide** if you want to inspect raw URLs in Excel.

5. **View Link List:** When a row has tracking URLs, the **View Link List** cell is a `#` in-sheet link; VBA
   collects AD–AR for that row, writes a temp UTF-8 `.txt` (one URL per line) and a matching
   `.ctx.tsv` with **Company**, **Order Number**, **Category**, etc. from that row, then runs
   `VIEWER=` from the ini **without showing a console window**. The **tracking link viewer** is a
   small grid (**#**, **Item**, **Description**, **Tracking link**): **single-click** expands the full
   URL in the last column; **double-click** opens the URL in your browser. Optional **Describe with
   OpenAI** uses `OPENAI_API_KEY` from the environment (set via Email Sorter Settings → `email_sorter_settings.json`; override model with `TRACKING_VIEWER_OPENAI_MODEL`).

You do **not** need to paste VBA by hand for normal use.

Requirements for auto-build
---------------------------

- **Microsoft Excel** (desktop) installed.
- **`pip install -r requirements.txt`** so **pywin32** is installed on Windows.
- The script briefly sets registry **AccessVBOM=1** (Trust access to the VBA project
  object model) while creating the template, then restores your previous setting.

If auto-build fails (Excel missing, COM error, Group Policy blocks VBOM), the script
falls back to **`orders.xlsx`** only when no valid macro template is available. With a working
template in `email_contents`, normal runs write **`orders.xlsm`**.

Open File Location helper
-------------------------

`copy_email_path_to_clipboard.py` decodes the `file:///…` URI and runs **`explorer.exe /select,<path>`**
(no extra Python packages for that step).

Trust macros when you open `orders.xlsm` the first time (or use a Trusted Location).

Updating the embedded macro after a code change
------------------------------------------------

The VBA lives inside `orders_template.xlsm`. After pulling a fix, delete
`email_contents/orders_template.xlsm` once and run createExcelDocument so Excel
automation rebuilds it from `macro_template.py` (or paste the latest `THISWORKBOOK_VBA`
into ThisWorkbook). This is required after VBA changes so **Open File Location** and **View Link List**
keep working; the generated `orders.xlsm` keeps whatever VBA was in the template when
it was loaded.

Manual fallback (only if automation failed)
--------------------------------------------

1. Create `orders_template.xlsm` in Excel (macro-enabled), sheet named **Orders**.
2. Alt+F11 -> **ThisWorkbook** -> paste the VBA from `macro_template.py` constant
   `THISWORKBOOK_VBA` (or copy from an older backup).
3. Save as `<BASE_DIR>/email_contents/orders_template.xlsm` (or your `EXCEL_TEMPLATE_PATH`).
4. Run createExcelDocument again.

Optional .env
-------------

  EXCEL_TEMPLATE_PATH=C:\full\path\to\orders_template.xlsm
  EXCEL_CLIPBOARD_INI_PATH=C:\full\path\to\excel_clipboard_launch.ini
  EXCEL_OUTPUT_PATH=C:\full\path\to\orders.xlsm

One-off: rebuild template only
-------------------------------

  python createExcelDocument/macro_template.py

(Loads `.env`; default template path is `<BASE_DIR>/email_contents/orders_template.xlsm`.)
