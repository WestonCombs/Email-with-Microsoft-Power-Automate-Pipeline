# Email Sorter (Python)

Email Sorter replaces a Power Automate flow with a local Python pipeline and desktop UI:

1. Sign in to Microsoft Graph and fetch email bodies/attachments
2. Extract structured order data with OpenAI
3. Track shipment links/numbers and cache 17TRACK status
4. Build and maintain an Excel workbook (`orders.xlsm` when macro template is available)
5. Support post-processing workflows (gift invoice linking, proof-of-delivery sync/capture, Excel user edits back to JSON)

`mainRunner.py` is the pipeline entry point. `email_sorter_launcher.py` is the main desktop launcher.

## Platform + Runtime

- Python **3.9+** (main runner enforces this)
- Windows is the primary supported platform for full feature set (Excel COM/VBA, tracking helper windows, assisted POD capture)
- Microsoft Excel desktop is required for macro-enabled workbook features

## Install

```bash
pip install -r requirements.txt
```

## Quick Start

1. Open the launcher:

```bash
python email_sorter_launcher.py
```

2. In **Settings**, configure:
   - `GRAPH_MAIL_FOLDER`
   - `AZURE_CLIENT_ID`
   - `AZURE_TENANT_ID` (optional override; defaults to `common`)
   - `OPENAI_API_KEY`
   - `SEVENTEEN_TRACK_API_KEY`
3. Click **Run** (or run `python mainRunner.py` directly).

## Main Pipeline (mainRunner.py)

Default run:

```bash
python mainRunner.py
```

Optional modes:

```bash
python mainRunner.py --skip-email-fetch
python mainRunner.py --custom-import-html
```

Pipeline stages:

1. `EnvironmentInitialization/runner.py`
   - Ensures required folders exist (attachments/html/pdf/json/log/state folders).
2. Microsoft Graph fetch (`emailFetching/ms_graph_fetcher.py`)
   - OAuth via MSAL (`interactive` or `device_code`)
   - Reads all messages from configured folder.
3. Extraction (`grabbingImportantEmailContent/grabbingImportantEmailContent.py`)
   - HTML -> plaintext normalization
   - Redirect resolution + tracking link classification
   - OpenAI extraction into canonical JSON fields
   - Gift-card follow-up classification logic
   - Optional HTML->PDF conversion fallback (`xhtml2pdf`) for problematic HTML files.
4. Sort (`sortJSONByOrderNumber/sortJSONByOrderNumber.py`)
5. Excel build/sync (`createExcelDocument/createExcelDocument.py`)

## Excel Workbook Features

Primary workbook output:

- `<BASE_DIR>/email_contents/orders.xlsm` when template/macros are available
- Falls back to `.xlsx` when macro template cannot be used

Macro/template support (`createExcelDocument/macro_template.py`):

- Builds/refreshes `orders_template.xlsm` through Excel COM
- Wires in `ThisWorkbook` VBA + hotkey module
- Writes `excel_clipboard_launch.ini` with paths to helper scripts

Workbook capabilities:

- Open file location links (internal `#` links + hidden URI columns + VBA follow handling)
- Tracking viewers launch from workbook columns:
  - View Tracking Links
  - View Tracking Numbers
  - View Tracking Numbers (All For Order)
  - Shipping Status (17TRACK viewer)
- Gift invoice workflow launch from `Invoice Link`
- Triple-Escape edit mode:
  - top-row rainbow cycle while armed
  - editable fields: company, total paid, tax paid
  - writes edits back to JSON via `createExcelDocument/excel_user_edit_sync.py`
  - persists overlay in `email_contents/json/excel_user_edits.json`
  - clearing an edited field restores original value and removes `*` marker

## Tracking + POD Tools

### Tracking link viewer

`trackingLinkViewer/tracking_link_viewer.py`

- Shows tracking URLs in a grid
- Saves per-row state (opened/delivered/cancelled)
- Supports context file (`.ctx.tsv`) generated by Excel VBA launch

### Tracking numbers viewer

`trackingNumbersViewer/tracking_numbers_viewer.py`

- `web` mode: per-number quick open to carrier page
- `order` mode: aggregated by tracking ID with frequency

### Shipping status viewer (17TRACK)

`trackingNumbersViewer/tracking_status_viewer.py`

- Smart cache-backed 17TRACK status lookups
- Row highlighting:
  - green: processed POD exists
  - gray: NotFound temporary/final retry state
- Right-click menu includes:
  - `Open` (plain browser open, no automation dependency)
  - `Delete` on processed rows (removes POD PDF + JSON references, marks as unprocessed again)
  - `Force Check Tracking Number`
- Assisted PDF capture controls (`Play`/`Pause`, toggle)

### POD data/workflow

`proofOfDelivery/pod_data.py` and `proofOfDelivery/pod_workflow.py`

- Maintains `proof_of_delivery.json`
- Watches open workbook and prompts to sync missing POD rows
- Supports Automation Hub “remaining PODs” launch mode
- Handles processed artifact deletion and viewer row model updates

## Authentication (Microsoft Graph)

Register an app in Azure / Entra ID and grant delegated `Mail.Read`.

Recommended:

1. App registration in a real Entra tenant
2. Redirect URI for public client (`http://localhost`)
3. Store client ID in launcher Settings
4. Use tenant that matches your account strategy (`common` for mixed accounts, tenant-specific when required)

Token cache default:

- `python_files/.graph_token_cache.bin`

## Notes for Maintainers

- If VBA changes, refresh template so generated workbook gets the new macro code:
  - `python createExcelDocument/macro_template.py`
- The workbook uses hidden columns and ini path cell (`AA1`) for helper launches; keep VBA constants and Python column constants aligned.
- Some legacy utilities remain for compatibility; active flows are centered on launcher + mainRunner + createExcelDocument + tracking/POD modules.
